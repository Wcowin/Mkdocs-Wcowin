## document-dates 插件 recently-updated 注入机制（排查首页排版乱）
- `recently-updated` 区块**仅靠 `<!-- RECENTLY_UPDATED_DOCS -->` 标记注入**（插件 `on_post_page` 对 `'\n<!-- RECENTLY_UPDATED_DOCS -->'` 做字符串替换），标记在哪，区块就插在哪。
- `mkdocs.yml` 里 `document-dates:` 下的 `position: bottom` 只影响「日期信息」位置，**不影响** recently-updated（recently-updated 没有自己的 position 配置）。
- 全站只有 `docs/index.md` 末尾用到该标记（`docs/blog/websitebeauty/mkdocs-document-dates.md` 里的是文档示例，在代码块内不触发）。
- 本地复现需装 `mkdocs-materialx` + `mkdocs-document-dates`（`python3.11 -m pip install`）。构建产物验证：区块落在 `</article>` 最末端。
- 首页曾因标记在页面中部（联系方式之前）导致区块注入中间、挤乱下方布局；已把标记移到 index.md 末尾。
- 擦亮标题 `.shine-text` 最终用双层 background（底层 180deg 抛光渐变 + 上层 110deg 细高光 48%/50%/52%），`background-position` 从 200%→0%（左→右）。

## 维护提示
- 改完 index.md 后必须重新 `mkdocs build`/重启 `mkdocs serve` 并硬刷，否则看到的是旧构建（曾误判为仍乱）。
